Understanding

What Masquerade Does

Masquerade takes a COBOL codebase and makes it understandable, testable, and migratable.

📄
IngestFeed in .cbl, .cpy, .bms, .jcl files from any COBOL system
↓
🗘
Parse & AnalyzeRecursive-descent parser extracts AST, data flows, conditionals, CICS ops
↓
📈
Dependency GraphCALL, COPY, FILE, CICS_IO edges across all programs, copybooks, screens
↓
📚
GenerateSpecs, skeletons (Python/Java/C#), behavioral tests, business rule catalogs
↓
✅
VerifyDifferential harness compares COBOL golden outputs to modern reimplementations field-by-field

Core Capabilities

  • Structural analysis — full AST with paragraphs, data flows, conditional logic, CICS operations
  • Python/Java/C# skeletons — typed stubs with copybook dataclasses, repository interfaces, API contracts
  • Behavioral test suites — pytest files generated from COBOL decision trees, not just stubs
  • Business rule extraction — evidence-anchored rules classified as ACCESS_CONTROL, ROUTING, VALIDATION, etc.
  • CobolDecimal arithmetic — faithful PIC-based precision, truncation, and rounding
  • Differential testing — field-by-field equivalence checking with confidence scoring
  • Impact analysis — "if this field changes, show the blast radius"
  • RAG-powered Q&A — ask natural-language questions about a COBOL codebase
Core Design Principle

Built for Coding Agents

Masquerade isn't just a tool for humans. It's designed from the ground up so that AI coding agents (Claude, Cursor, Copilot, Devin, etc.) can operate on legacy COBOL codebases autonomously.

Why This Matters

Most COBOL tooling outputs prose reports, PDFs, or GUI dashboards. An agent can't use any of that. Masquerade outputs structured, machine-readable JSON at every stage — so an agent can parse a COBOL system, read the dependency graph, understand the business rules, generate a reimplementation, and verify it all passed — in a single automated loop.

This makes Masquerade the COBOL-aware toolkit layer that any coding agent can build on top of.

📄

Structured JSON Outputs

Every pipeline stage produces machine-readable JSON — programs.json, graph.json, stats.json. No prose, no PDFs. Agents can parse and reason over results directly.

⌨

CLI-Native Interface

Single-command analysis: python analyze.py <path>. No GUI required, no interactive prompts. Agents can invoke the full pipeline from a shell command.

🔧

Deterministic Pipeline

Same input, same output. No randomness in parsing, graph building, or skeleton generation. Agents can rely on stable, reproducible results across runs.

🔍

Typed Data Contracts

Copybook fields carry PIC metadata (digits, scale, signed, usage). Generated skeletons use typed dataclasses, not string blobs. Agents get real type information to work with.

✅

Self-Verifying Loop

The differential harness lets an agent write a reimplementation and immediately verify it against golden COBOL output — no human in the loop needed. Fail, fix, re-run.

📊

Readiness Scoring

Every program gets a numeric readiness score (isolation, simplicity, testability). Agents can prioritize which programs to tackle first without needing human judgment.

The Agent Workflow

A coding agent working with Masquerade follows this loop:

# 1. Agent analyzes the COBOL codebase
python analyze.py ../test-codebases/carddemo

# 2. Agent reads structured output
#    programs.json  → AST, data flows, conditionals per program
#    graph.json     → dependency graph (who calls whom)
#    stats.json     → LOC, complexity, readiness scores

# 3. Agent picks highest-readiness program and reads its spec
python cli.py generate --spec CBACT01C

# 4. Agent generates skeleton + tests
python cli.py generate --skeleton CBACT01C --lang python
python cli.py generate --tests CBACT01C

# 5. Agent writes the reimplementation (filling in the skeleton)
# 6. Agent runs differential tests to verify equivalence
python -m pytest tests/test_differential_full.py -v

# 7. If tests fail → agent reads the diff report, fixes, re-runs
# 8. If tests pass → move to next program

Prep Work

Clean up the codebase before anyone sees it.

  • Audit for secrets and credentials Grep for API keys, tokens, passwords, .env files, Pinecone keys, Gemini keys. Remove or rotate ALL of them. Check git history too — use git filter-repo if needed.
  • Clean up .gitignore Ensure .env, __pycache__, .runtime/, *.egg-info, node_modules, and any local config are excluded.

  • Remove internal/vendor-specific references Scrub any references to specific clients, internal project names, Jira tickets, or private URLs from comments and docs.
  • Pin dependencies with versions Ensure requirements.txt has pinned versions. Add a pyproject.toml if not already present for modern Python packaging.
  • Run the full test suite — fix any failures cd pipeline && python -m pytest tests/ -v — all 37 test files must pass cleanly.
  • Decide what to include vs. exclude The services/ directory (FastAPI microservices) and infra/ (Terraform/K8s) may be too early-stage. Consider starting with just pipeline/ and showcase/.

Step 2

Choose a License

Your license signals your intent and shapes how people can use your work.

Recommended: Apache 2.0

Best for this project because:

  • Enterprise-friendly — banks and insurers can adopt without legal friction
  • Patent grant protects contributors and users
  • Compatible with most other OSS licenses
  • Doesn't prevent you from building a commercial offering on top later

Alternatives to Consider

  • MIT — simpler, maximally permissive, but no patent protection
  • AGPL-3.0 — if you want to force cloud users to open-source modifications (SaaS protection). Enterprises may avoid it.
  • BSL (Business Source License) — open-source with delayed commercial use. Used by Sentry, CockroachDB. Good if you want to monetize directly.
  • Pick your license and add the LICENSE file Replace the current LICENSE with the full text. Add SPDX identifier to pyproject.toml.
  • Add license headers to source files At minimum, add a one-liner to each .py file: # SPDX-License-Identifier: Apache-2.0
  • Check dependency license compatibility Run pip-licenses to list all dependency licenses. Flag any GPL/AGPL contamination.
